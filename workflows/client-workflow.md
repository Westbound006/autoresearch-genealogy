# Client Workflow

How to set up a new client project and produce annotated document deliverables using autoresearch-genealogy.

## Overview

Each client gets their own Obsidian vault (or a dedicated folder within their vault). This keeps research, documents, and notes fully separated across clients. The `setup-client-vault.sh` script handles deployment; the prompts handle the research.

## Step 1: Set Up the Client Vault

Run the setup script once per client, pointing it at the client's existing vault folder:

```bash
./setup-client-vault.sh /path/to/ClientVault "Smith Family"
```

This copies the vault template into `/path/to/ClientVault/Genealogy/` and substitutes `[CLIENT]` with `"Smith Family"` throughout all template files.

If the client's vault does not exist yet, create the folder first:

```bash
mkdir -p /path/to/ClientVault
./setup-client-vault.sh /path/to/ClientVault "Smith Family"
```

Open the vault in Obsidian and verify the folder structure:

```
ClientVault/
└── Genealogy/
    ├── _Index.md
    ├── Family_Tree.md
    ├── Open_Questions.md
    ├── Research_Log.md
    ├── Data_Inventory.md
    ├── Timeline.md
    └── templates/
        ├── person.md
        ├── transcription.md
        ├── certificate.md
        ├── postcard.md
        └── region.md
```

## Step 2: Enter What the Client Provided

Fill in `Family_Tree.md` with whatever the client has shared: names, dates, locations, oral history. Even rough information is useful.

Update `Data_Inventory.md` with any documents the client supplied (originals, scans, photos). Note file paths for any scans.

## Step 3: Annotate the Client's Documents

If the client has scanned documents (certificates, letters, postcards, photos), run the document annotation prompt:

1. Open Claude Code in the `Genealogy/` folder
2. Paste the contents of `prompts/13-document-annotation.md`
3. Replace the placeholders:
   - `[SCAN_PATH]` with the folder containing the client's scans (e.g., `~/Files/Genealogy/SmithFamily/`)
   - `[VAULT_PATH]` with the path to `Genealogy/` inside the client vault
   - `[CLIENT]` with the client name (e.g., `Smith Family`)
4. Let it run

After completion, the vault will contain a structured transcription note for each document, plus a batch summary index. All notes will have `client: Smith Family` and `deliverable: true` in their frontmatter.

## Step 4: Expand and Verify the Tree

Once documents are annotated, run the research prompts:

| Prompt | Purpose | When to run |
|---|---|---|
| `01-tree-expansion.md` | Find parents and siblings for every ancestor | After entering baseline data |
| `02-cross-reference-audit.md` | Fix date and name discrepancies | After tree expansion |
| `08-open-question-resolution.md` | Resolve specific gaps the client raised | Any time |

For each prompt, replace `[VAULT_PATH]` with the client vault's `Genealogy/` path and `[ANCESTOR]` / `[SURNAME]` with the relevant names.

## Step 5: Review Before Delivering

Before handing anything to a client:

1. Open each note with `deliverable: true` and read it through
2. Change `reviewed: false` to `reviewed: true` once you have checked it
3. Check that every claim in the Commentary section is labeled as fact or inference
4. Verify that illegible portions are marked `[unclear]`, not guessed at

To find all unreviewed deliverable notes in Obsidian using the Dataview plugin:

```dataview
TABLE file.name, document_type, person, ocr_quality
FROM ""
WHERE deliverable = true AND reviewed = false AND client = "Smith Family"
SORT file.name ASC
```

## Step 6: Deliver

What to hand over to the client:

- **Annotated document notes**: all notes with `deliverable: true` and `reviewed: true`
- **Document batch index**: the `Document_Batch_[date].md` file generated by prompt 13
- **Family tree**: `Family_Tree.md` with all expansions applied
- **Timeline**: `Timeline.md` if populated

What to keep internal (not for the client):

- `Research_Log.md`: your working search log; contains negative results and process notes
- `Open_Questions.md`: your research gaps list
- `Data_Inventory.md`: your file management notes

The simplest delivery method is to export the relevant notes as PDFs from Obsidian (File > Export to PDF) or to share the vault folder directly if the client uses Obsidian themselves.

## Managing Multiple Active Clients

- Keep each client in a separate vault or a clearly separated top-level folder
- Never mix client data: the `client` frontmatter field is a safety net, not a substitute for separation
- The `client` field enables cross-vault queries if you ever consolidate into one vault for internal tracking
- Name vault folders consistently: `[ClientSurname]_[Year]` (e.g., `Smith_2026`)

## Placeholders Reference

| Placeholder | Replace with |
|---|---|
| `[CLIENT]` | Client name (e.g., `Smith Family`) |
| `[VAULT_PATH]` | Full path to the `Genealogy/` folder inside the client vault |
| `[SCAN_PATH]` | Full path to the folder containing the client's scanned documents |
| `[SURNAME]` | Primary research surname |
| `[ANCESTOR]` | Specific ancestor's full name |
| `[LOCATION]` | Town, county, or country of origin |
| `[DATE]` | Specific date or year range |
